# LLM Output Evaluation Rubric

This rubric is designed to assess the quality of outputs generated by an AI agent evaluating Big4Bot consulting frameworks.

## 1. Schema Conformance (Pass/Fail)
- **Pass:** The output perfectly matches the expected `output_schema.json` format, including correct types and required fields.
- **Fail:** The output deviates from the schema structure, uses incorrect types, or includes unauthorized JSON blocks or hallucinated fields.

## 2. Methodological Adherence (1-5)
- **5 (Excellent):** The agent followed the step-by-step diagnostic process in `SKILL.md` exactly, showing clear progression between logic steps.
- **3 (Acceptable):** The agent followed the core methodology but skipped minor nuances or blended steps together.
- **1 (Poor):** The agent hallucinated a different methodology entirely or ignored the requested analytical boundaries.

## 3. Data Integration & Synthesis (1-5)
- **5 (Excellent):** Inputs provided in `input_schema.json` are woven deeply into the output. Assumptions are explicitly stated if data is missing.
- **3 (Acceptable):** Inputs are referenced but not deeply explored, creating superficial insights.
- **1 (Poor):** Provided inputs are ignored; the agent relies on generic training data rather than the user's explicit context.

## 4. Analytical Depth & Insight (1-5)
- **5 (Excellent):** The output reads like professional consulting tier work, highlighting non-obvious root causes or second-order effects.
- **3 (Acceptable):** The output is logically sound but basic, largely summarizing what is already known.
- **1 (Poor):** The output is contradictory, trivial, or dangerously simplistic.
